# Troubleshooting Methodology

This structured approach is used for all lab simulations.

## 1. Identify the Problem
- What is the user reporting?
- What system is affected?
- When did it start?

## 2. Establish a Theory of Probable Cause
- Network issue?
- Permission issue?
- Software malfunction?
- User error?

## 3. Test the Theory
- Run diagnostic commands
- Review logs
- Restart services
- Attempt alternate login

## 4. Establish Plan of Action
- Reset password
- Reconfigure network
- Reinstall application
- Adjust permissions

## 5. Verify Full System Functionality
- Confirm issue resolved
- Test affected system
- Ensure no new issues created

## 6. Document Findings
- Root cause
- Resolution
- Prevention steps
